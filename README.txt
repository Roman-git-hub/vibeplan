Что важно:
- Данные (state) живут ТОЛЬКО в памяти и в файле в вашем репозитории (data.json).
- Никаких долгоживущих данных в localStorage, кроме: настройки GitHub (owner/repo/path/token) и флага 'запомнить вход'.
- Синхронизация только вручную: кнопка 'Синхронизировать' — записать текущую память в GitHub;
  кнопка 'Загрузить с GitHub' (в настройках) — загрузить данные из репозитория и заменить текущую память.
- Вход: при первом запуске просит логин/пароль; если вы отметите 'Запомнить вход', он будет храниться в настройках (localStorage) до выхода.

Инструкции:
1. Создайте Personal Access Token (classic) в GitHub с правом public_repo (или repo для приватного).
2. Вставьте owner/repo/path/token в Настройки на сайте.
3. Нажмите 'Загрузить с GitHub' чтобы подтянуть существующие данные (если есть).
4. Редактируйте, затем нажмите 'Синхронизировать' чтобы сохранить в репозитории.
